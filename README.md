#java-completable-future-patterns

## Project Goal (Мета Проєкту)

Цей проєкт був розроблений для поглибленого вивчення та демонстрації принципів **асинхронного програмування** в Java, зосереджуючись на створенні гнучких, залежних ланцюжків виконання та ефективному паралельному об'єднанні результатів за допомогою класу **`java.util.concurrent.CompletableFuture`**.

Він охоплює два різних, але взаємодоповнюючих, асинхронних сценарії: послідовне виконання залежних операцій та одночасне виконання незалежних завдань з подальшою агрегацією результатів.

-----

## Implementation Details (Деталі Реалізації)

Проєкт реалізує дві окремі задачі (Task 1 та Task 2), кожна з яких розміщена у власному пакеті та має свій головний клас (`main`).

### Key Technologies and Concepts (Ключові Технології та Концепції)

| Технологія / Концепція | Призначення |
| :--- | :--- |
| **Java Development Kit (JDK) 8+** | Базова платформа. |
| **`CompletableFuture`** | Основний клас для побудови асинхронної логіки. |
| **`thenCompose()`** | Використовується для створення **залежного ланцюжка** (pipeline), де результат першого Future є вхідним параметром для запуску **нового** асинхронного Future (Завдання 1). |
| **`allOf()`** | Використовується для **паралельного виконання** групи незалежних завдань і очікування на завершення **всіх** з них (Завдання 2). |
| **`anyOf()`** | Використовується для очікування завершення **першого** завдання з групи (Завдання 2, демонстрація). |
| **`thenApply()` / `getNow()`** | Використовуються для перетворення результатів та безпечного вилучення даних після завершення `allOf()` (Завдання 2). |
| **`supplyAsync()`** | Створення асинхронних завдань, які **повертають результат**. |
| **`exceptionally()`** | Обробка винятків у ланцюжку `CompletableFuture` без переривання виконання головного потоку. |
| **Імітація затримки** | Використання `TimeUnit.SECONDS.sleep()` для імітації мережевих запитів або доступу до бази даних. |

## Task Descriptions (Опис Завдань)

### 1. Ланцюжок Залежностей (Пакет: `task1_chained_future`)
* **Метод:** `thenCompose()`
* **Сценарій:** Асинхронне завдання 1 отримує ID користувача, а завдання 2, яке залежить від цього ID, використовує його для отримання детального профілю.

### 2. Паралельне Комбінування (Пакет: `task2_combined_futures`)
* **Методи:** `allOf()`, `anyOf()`
* **Сценарій:** Паралельно отримуються погодні дані (температура, вологість, вітер) для трьох різних міст. Після завершення всіх запитів дані агрегуються, порівнюються та надаються висновки щодо рекомендацій одягу/відпочинку.

Для запуску кожного завдання перейдіть до його відповідної директорії та виконайте команди компіляції та запуску.

### Завдання 1: Ланцюжок Залежностей

```bash
cd task1_chained_future/
javac DatabaseProcessor.java
java DatabaseProcessor
````

### Завдання 2: Паралельна Обробка

```bash
cd task2_combined_futures/
javac WeatherComparator.java
java WeatherComparator
```
